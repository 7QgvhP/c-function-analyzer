# 変更履歴 (Change Log)

本プロジェクト（C Function Analyzer）のすべての変更は、このファイルに記録されます。

## [1.13.4] - 2026-06-27
### 修正
* グローバル変数として定義された配列や構造体配列（例: `hogestruct[0].a = 100;`）への書き込み（代入）が、出力変数（Outputs）に分類されず検出漏れしていた不具合を修正。

## [1.13.3] - 2026-06-27
### 修正
* 多次元配列、ネストされた構造体、多重ポインタなどへの書き込み（代入）が「出力変数」に正しく検出されない不具合を、構文木を再帰的に解決するロジックを導入して修正。
* 複合代入演算子（`+=` など）およびインクリメント・デクリメント（`++`, `--`）を伴う書き込みに対し、「入力変数」としてもカウントするよう判定ロジックを改善。
* グローバル変数スキャン時にインライン構造体定義が含まれる場合、余分な `{...}` 部をカットして型名バッジの表示崩れを防ぐよう改善。
* Webview 上に埋め込むすべての変数名・型名に HTML エスケープ処理（XSSおよび表示崩れ対策）を追加。
* エディタ上に実体のない `戻り値 (return)` などの項目がクリックされた際、不要なハイライト検索をスキップする早期リターン処理を追加。

## [1.13.2] - 2026-06-25
### 修正
* `static void` や `inline void` のような修飾子付き void 戻り値関数を解析した際に、出力変数リストに誤って「戻り値 (return)」が追加表示されてしまう不具合を修正。

## [1.13.1] - 2026-06-18
### 変更
* 解析結果画面（Webview）のヘッダー部分から、関数名横に表示されていた戻り値の型名表示を削除。

## [1.13.0] - 2026-06-17
### 追加
* ファイル直下の変数宣言を事前にスキャンし、ファイル内定義のグローバル変数の型情報（`int` や `float` など）を自動適用してWebviewの型名バッジに表示する機能を追加。
  * ファイル内で定義が見つからないグローバル変数については、従来通り `global (推定)` として表示します。

## [1.12.1] - 2026-06-17
### 変更
* 推定されたグローバル変数の型表示エリアの文言を `extern/global (推定)` から `global (推定)` に変更。

## [1.12.0] - 2026-06-17
### 変更
* 変数リストの表示順序を「変数名 ➔ 型名」から「型名 ➔ 変数名」に入れ替え。
* 型名バッジの幅を固定（`140px`）に統一し、変数名の左端が縦に綺麗に揃うようにレイアウトを改善。

## [1.11.1] - 2026-06-17
### 修正
* 代入式の左辺（書き込み先）として使われているグローバル変数やポインタ引数が、誤って「入力変数（読み取り）」にも分類されて重複表示されてしまうバグを修正（Tree-sitterノードのオブジェクト比較方法のバグを修正）。

## [1.11.0] - 2026-06-16
### 追加
* 各変数および呼び出し関数の右側に個別の「コピー」ボタンを追加。クリックすると変数名または関数名のみ（例: `hoge`）を直接クリップボードにコピーできる機能を追加。
* 個別コピーボタンは、変数行にホバーした際にのみ表示されるスタイリッシュなUI設計に。
### 変更
* カード（カテゴリ見出し）の右側に配置されていた「一括コピー」ボタンをすべて削除。

## [1.10.1] - 2026-06-16
### 修正
* Webview 内のセキュリティ制約（セキュアコンテキスト制限）により、一括コピーボタンを押した際にクリップボードへのコピーが機能しない不具合を修正。コピー処理を Webview から VS Code 本体（拡張機能ホスト側）の API にメッセージ委譲するように改善。

## [1.10.0] - 2026-06-16
### 追加
* 各カテゴリ（入力変数、出力変数、内部変数など）のカードの右上に、そのカテゴリ内の変数リストを一括でクリップボードにコピーできる「コピー」ボタンを追加。
* コピー成功時に「コピー完了」と表示し、一時的にボタンをテーマカラーで強調するインタラクティブなフィードバックを追加。

## [1.9.0] - 2026-06-16
### 追加
* 各分類の見出しの右側に、そのカテゴリに検出された変数や関数の総数を示すカウント数バッジを表示する機能を追加。

## [1.8.0] - 2026-06-16
### 変更
* 変数が「入力」と「出力」の両方の条件に合致する場合（読み取りと書き込みが両方行われているポインタ引数やグローバル変数）、これまでは「出力」のみに表示されていた仕様を変更し、「入力変数」と「出力変数」の両方のリストに重複して表示されるように改善。

## [1.7.1] - 2026-06-16
### 修正
* Webview を閉じることなく複数の関数を連続で解析した際、変数を選択したときのハイライト対象が直近の関数ではなく、最初に解析した関数の行範囲のままになってしまうデータ同期バグを修正。
### 変更
* 各分類の見出し下部に表示されているカテゴリ別のアクセントラインの長さを、見出し全体の幅（100%）に引き伸ばし。

## [1.7.0] - 2026-06-15
### 変更
* 解析結果画面（Webview）のUIデザインを、視覚的アクセントを施したモダンなカード型レイアウトに刷新。
* カテゴリごとに個別のテーマカラーを設定し、情報のスキャナビリティ（視認性）を大幅に向上。
* 変数名の下に表示されていた分類詳細（例: 「入力引数（値渡し）」など）を削除し、UIをよりシンプルに整理。
* 変数の型名表示を、薄い半透明背景を用いた「バッジ」スタイルに統一。
* 変数や関数名にマウスホバーした際、右へ滑らかにスライドするマイクロインタラクション（ホバーエフェクト）を追加。
* データ未検出時の表示エリア（プレースホルダー）を、クリーンな破線枠のデザインに改善。

## [1.6.0] - 2026-06-15
### 追加
* Webview 上の変数名をクリックした際、コード上の配列（例: `hoge[i]`）や構造体（例: `hoge.A`, `ptr->member`）全体のアクセス表現もカバーして強調表示（ハイライト）される機能を追加。

## [1.5.2] - 2026-06-14
### 修正
* 関数ポインタ変数（例: `math_func`）の宣言を正しくパースして「内部変数（ローカル変数）」に分類されるように修正。
* 関数ポインタによる呼び出しが「入力変数」に誤検出されてしまう不具合を修正。
* 関数ポインタによる呼び出しを「呼び出し関数」リストから除外。

## [1.5.1] - 2026-06-14
### 修正
* Webview からのフォーカス移動時に、エディタ上の強調表示（ハイライト）が即座にクリアされてしまう不具合を修正。
* Webview 操作時にアクティブエディタが見失われ、強調表示（ハイライト）が適用されない問題を修正（複数エディタから解析対象ファイルを正しく探索するように改善）。

## [1.5.0] - 2026-06-14
### 追加
* 解析結果画面（Webview）上の変数名・関数名をクリックした際に、コード上の該当シンボルを一時的に強調表示（ハイライト）し、画面中央へ自動スクロールする機能を追加。
* エディタ上でカーソルを移動させたりテキストを編集したりした際に、強調表示（ハイライト）が自動で解除される機能を追加。

## [1.4.1] - 2026-06-14
### 変更
* 解析結果画面（Webview）における関数名・マクロ関数名の表示デザイン（レイアウト・下破線・ホバーエフェクト等）を変数名と統一。

## [1.4.0] - 2026-06-14
### 変更
* 解析結果画面（Webview）の各分類見出しから英語表記（`(Inputs)` など）を削除。
* 解析結果画面（Webview）の表示順序を、「呼び出し関数」の直後に「マクロ関数」が来るように変更。

## [1.3.0] - 2026-06-14
### 追加
* 変数名または関数名がすべて大文字のとき、通常の入出力や呼び出し関数とは別の分類として「マクロ変数 (Macro Variables)」および「マクロ関数 (Macro Functions)」のセクションに表示する機能を追加。
* 上記の大文字マクロ分類機能のオン・オフを切り替える設定項目（`c-function-analyzer.classifyAllUppercaseAsMacros`）を VS Code 設定に追加。

## [1.2.0] - 2026-06-09
### 変更
* プロジェクト内の識別子・コマンドID表記を `c-function-analyzer` に統一。
* 解析結果画面（Webview）のレイアウトを、ウィンドウ幅に関わらず常に縦1列で表示するように変更。
* 不要な開発ドキュメント（`requirements.md`、`implementation_plan.md`）を削除。

## [1.1.1] - 2026-06-06
### 修正
* 配列引数（例: `int a[]`）に対して配列書き込み（例: `a[5] = 10;`）を行った際、入力変数として誤認識されたり、グローバル変数（出力）として誤検出されたりするバグを修正。
* ポインタ引数の型表記において、型名に `*` が二重に付与されてしまう問題（例: `int **`）を修正。

## [1.1.0] - 2026-06-06
### 変更
* 解析結果画面（Webview）のUIを、VS Codeの標準テーマに適合するシンプルなフラットデザインに変更。

## [1.0.1] - 2026-06-06
### 修正
* `web-tree-sitter` のインポート処理を CommonJS（CJS）向けに名前付きインポートへ修正し、`init is not a function` のエラーを解決。
* `web-tree-sitter` を `0.20.8` にダウングレードし、WASM（C言語パーサー）との ABI バージョン不一致によるロードエラーを解決。
* パッケージ（VSIX）内の不要な旧 WASM ファイル（`web-tree-sitter.wasm`）および開発用デバッグログを削除（パッケージサイズを約 288 KB へ削減）。

## [1.0.0] - 2026-06-05
### 追加
* 初期リリース（C言語関数の静的解析拡張機能）。
* `web-tree-sitter` による AST 解析の実装（引数・ポインタ出力・グローバル変数・直接関数呼び出しの抽出）。
* VS Code テーマ適応型の Webview 解析結果画面の提供。
* ショートカットキー `Ctrl + Alt + A` での起動サポートを追加（C言語エディタ用）。
